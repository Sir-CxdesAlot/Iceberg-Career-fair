layer 03 · deeper truths
The Codebase
Dark Side
naming conventions
Variables named data2, dataFinal, dataFinalActual, USE_THIS_ONE.
// all in the same file
legacy code
A function called doTheThing(). Nobody touches it. The company runs on it.
// written in 2011, author has left the building
testing strategy
"We test in production." said with complete confidence.
// users are unpaid QA engineers
the magic number
There's a 0.00743 somewhere deep in the code. Nobody knows where it came from. Removing it breaks everything.
// it's load-bearing math
the sacred if-statement
if (user === "Dave") — Dave left in 2018. The condition is still there. Dave must never be logged in.
// nobody knows what happens if Dave logs in
commented-out code
A 200-line block commented out since 2015. Nobody deletes it. It's basically a historical monument.
// "might need it later" — someone, 9 years ago

infrastructure

layer 05 · the abyss approaches
Production
Horrors
deployment
Friday 5pm deploy. "What's the worst that could happen?"
// you find out at 2am Saturday
the database
A spreadsheet. In Google Drive. Shared with the whole company. With edit access.
// this is the source of truth
monitoring
You find out the server went down when a client tweets about it.
// alerts were set up, nobody got them
race condition
Two parts of the code both try to update the same thing at the same time and fight each other. Like two people trying to go through a door simultaneously. In prod. At 3am.
// turns out order matters. nobody told the threads.
memory leak
The app quietly eats RAM like a goldfish that never stops growing until the server runs out of memory and collapses. Nobody notices for 6 weeks.
// "why is the server so slow on Mondays"
the hotfix
Deployed a "quick hotfix" at 11pm. The hotfix needed a hotfix. The second hotfix broke something else. It's 4am now.
// this is called a hotfix chain. it has no end.
layer 06 · the bottom of the iceberg
The Forbidden
Knowledge
the senior dev secret
"I'm not smarter than you. I've just seen this exact bug before."
// experience is just a graveyard of failed deploys
the final truth
Nobody fully understands the system. We're all just guessing confidently.
// this includes the people who built it
career advice
The best code is the code you didn't write.
// now go build something amazing anyway
undefined behaviour
The program does something the language spec says should not be possible. It works anyway. On Tuesdays. Only in Germany. Nobody ships a fix because it might stop working.
// "don't touch it. it knows."
Heisenbugs
A bug that disappears the moment you try to debug it. It only exists when nobody is watching. Like a shy gremlin living inside the RAM.
// named after Heisenberg. yes, the physicist. yes, this is real.
the void
You've been staring at the same semicolon for 45 minutes. You are no longer debugging. You are becoming one with the error.
// take a walk. it'll fix itself. it always does.
